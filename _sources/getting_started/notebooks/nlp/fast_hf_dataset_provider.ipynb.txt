    "# Rapid NLP-based Data Creation/Encoding using Memory Mapping\n",
    "\n",
    "In this notebook, we will use a fast dataset provider-based abstraction that interfaces with Hugging Face's `datasets` (and has been created by HazyResearch). The key advantage of this approach is the use of either shared memory or memory maps in Python to accelerate the caching process. Furthermore, the dataset is cached as a contiguous numpy array, enabling manipulation of data with any sequence length. This feature eliminates the need for re-encoding data for multiple lengths, streamlining the data processing pipeline."

    "## Instantiating the Provider\n",
    "\n",
    "The first step is to instantiate the `FastHfDatasetProvider.from_hub()`, which loads and encodes the dataset. A set of arguments can be passed to its class method according to the user's needs:\n",
    "\n",
    "* `dataset_name`: Name of the dataset.\n",
    "* `dataset_config_name`: Name of the dataset configuration.\n",
    "* `data_dir`: Path to the data directory.\n",
    "* `tokenizer`: Instance of tokenizer to use.\n",
    "* `tokenizer_name`: Name of the tokenizer, if `tokenizer` has not been passed.\n",
    "* `tokenizer_max_length`: Maximum length of the tokenized sequences.\n",
    "* `mapping_column_name`: The columns in `dataset` that should be tokenized.\n",
    "* `validation_split`: Fraction of the dataset to use for validation.\n",
    "* `seed`: Random seed.\n",
    "* `num_workers`: Number of workers to use for encoding.\n",
    "* `use_eos_token`: Whether to use EOS token to separate sequences.\n",
    "* `use_shared_memory`: Whether to use shared memory for caching.\n",
    "* `cache_dir`: Path to the cache directory."
